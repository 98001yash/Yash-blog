## Problem

The operations team relied on disconnected dashboards for uptime, customer impact, and product performance. Context switching made incident response slower and blurred accountability during high-pressure moments.

## Solution

I designed a consolidated dashboard with role-aware summaries, drill-down panels, and a consistent interaction model for alerts, timelines, and metric exploration.

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Recharts
- Zod
- Vercel Analytics

## Architecture

The interface used a layered architecture: route-level data composition, reusable visualization modules, and domain-specific feature slices that could evolve independently. Shared data contracts kept the UI predictable.

## Challenges

The main challenge was balancing high information density with scannability. We needed to preserve power-user workflows without overwhelming occasional users during incidents.

## Engineering Decisions

I chose server-rendered route shells with progressively enhanced client-side visualizations. That kept first load fast while still enabling interactive drill-down experiences where they mattered most.
